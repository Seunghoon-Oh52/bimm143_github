Background

All functions in R have at least 3 things:

  • a name (we pick that and use it to call the function),
  • input arguments (one or more comma separated inputs that go inside the brackets when we call the function),
  • the body (the lines of R code that do the work of the function)

A first function

Here we will create a function to add some numbers. Let’s call it add().

Input arguments can be either “required” or “optional”. The latter have fall-back default values that will be used if the user does not specify them.

add1 <- function(x, y=0) {
  x+y
}

Q2b: Your second version should optionally be able to return either a multi-element vector of single character nucleotides (as before) or a single character string (not a vector of single letters but a singe vector of multiple letters). For example “AAGGCTG”. [1 pt]

Functions that could be useful here are paste(), if(), and return().

generate_dna_2b <- function(length_dna, single_string=FALSE){
  result_vec <- sample(x=c("A","C","G","T"), size=length_dna, replace=TRUE)
  
  if (single_string){
    return(paste(result_vec, collapse=""))
  }
  else {
    return(result_vec)
  }
}

Q2c: Finally, create a final version of your function that prints out a FASTA format sequence with an id line indicating the sequence length.

I created the version to automatically print out the FASTA format if thesingle_string argument is set as TRUE.

generate_dna_2c <- function(length_dna, single_string=FALSE, FASTA=TRUE){
  result_vec <- sample(x=c("A","C","G","T"), size=length_dna, replace=TRUE)
  
  if (single_string){
    result_fastaseq <- paste(result_vec, collapse="")
    cat(">len",length_dna,"\n", result_fastaseq, sep="")
  }
  else {
    return(result_vec)
  }
}

And the example result is:

CTGAAGTGG

A generate_protein() function

Q3. Write a function generate_protein() that returns a random peptide/protein sequence of a length specified by the user. For example generate_protein(6) might return “WQRTAG”.

The function first generates the result_protein vector by sampling among 20 amino acids with the given size argument (the length). replace argument in the sample() should set as “TRUE”, since same amino acids may be present in this amino acid sequence for the protein. Then we add up all the vector components - the single characters - into a single sequence string.

generate_protein <- function(length_protein){
  result_protein <- sample(x=c("A", "R", "N", "D", "C", "E", "Q", "G", "H", "I", "L", "K", "M", "F", "P", "S","T", "W", "Y", "V"), size=length_protein, replace=TRUE)
  return(paste(result_protein, collapse=""))
}
generate_protein(6)
[1] "RLFFIP"

Q5b: Speculate why very short random peptides are almost always found in nr, while longer ones typically are not. Your answer should refer both to the size of the sequence space (\(20^L\) for a peptide of length 𝐿) and to the size of the known protein universe. [1 pt]

: The size of the sequence space increases exponentially as the peptide length L increases, while the size of the know protein universe is finite: in other words, when we are using BLASTp to compare our generated peptide sequence to the know Non-redundant protein sequences (nr) database, at some point (length) the sequence space size exceeds the size of the nr database. This makes the probability of our peptide being not unique to exponentially decrease, and most of the peptides above certain length will be ~100% unique from the BLASTp results. Thus very short random peptides, having a smaller (or similar) sequence space compared to nr database, are always found in nr, while longer ones are mostly found unique, and the threshold of this “space size order reversal” seems to lie around the length (L) 8-9 from my results.

Connecting your findings to immunology (MHC class II and T-cell activation)

Q6: In 3–6 sentences total and using your Q5 data and the reasoning from Q5b, what do you think this minimum length is and why might it be a bad design choice for the immune system to present very short peptides? [3 pt]

: From the answer for Q5b, I estimated the threshold length for “sequence space size order reversal” to be around 8-9 from my data. Since MHC-presented peptides should be distinguished by the T-cell receptor if it’s self or non-self, the minimal length should be set to make the peptide unique from each pool of “self” and “non-self” peptides. If the displayed peptide length is too short, the receptors might misconceive the self peptide as non-self, or vice versa - since the peptides are not unique.
